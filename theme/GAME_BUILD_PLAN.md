# 미완성 게임 빌드 기획서

> 새 세션에서 이 파일 읽고 **순차적으로** 빌드 시작 (Claude↔Codex 번갈아).
> 완성 기준: emotion_mirror(774줄), number_frog(894줄), market_day(1172줄) 참고

## 퀄리티 기준 (최우선)

게임 수보다 **퀄리티**가 핵심. 아이가 자연스럽게 빠져들 수 있어야 함.

### 중독성 루프 설계 원칙
- **즉각 피드백**: 탭/클릭 순간 시각+물리 반응 (0.1초 이내)
- **적절한 긴장감**: 타이머, 하트, 점수 중 최소 1개 — 완급 조절
- **보상감**: 정답 시 파티클·사운드·캐릭터 반응이 확실히 터질 것
- **"한 판 더" 유도**: 라운드 끝에 점수가 아깝게 느껴지도록 설계
- **난이도 곡선**: 처음 2~3라운드는 누구나 성공 → 점점 자연스럽게 어려워짐

### 시각 퀄리티 기준
- 애니메이션: CSS transition/keyframe 풍부하게 (딱딱한 순간이동 금지)
- 캐릭터: SVG에 눈·입·표정 변화, 정답/오답 시 다른 반응
- 이펙트: 정답=파티클 폭발, 오답=흔들림+빨간 플래시
- 배경: 정적이지 않고 살아있는 느낌 (별 반짝임, 배경 요소 미세 움직임)
- 색상: 채도 높고 대비 강한 유아용 팔레트 사용

### 조작감 기준
- 탭 영역: 실제 시각 요소보다 20px 이상 크게 (미스터치 방지)
- 드래그: snap 애니메이션 필수, 놓는 순간 "딱!" 느낌
- 버튼: hover/active 상태 scale 변화로 눌리는 느낌

## 공통 규칙 (모든 게임)

- 기술: 순수 HTML + CSS + JS (zero dependencies)
- 테마: 다크 스페이스 `#0a0118` 배경, 별 55개 반짝임
- 폰트: Nunito (Google Fonts CDN)
- 이모지 금지: 모든 시각 요소 SVG로 직접 드로잉
- HUD: 좌상단 ← 홈 버튼 (`../../index.html`), 우상단 하트 3개 또는 진행 표시
- 화면: 800×600 고정 game area, 뷰포트 중앙 정렬
- 언어: 한국어 전용
- 결과 화면: 별점/점수 + 다시하기 + 홈 버튼

- 친구랑 싸웠을 때 → 슬픔/화 (10~40)

- 실수로 물건 떨어뜨렸을 때 → 놀람/당황 (20~60)

### 11. robot_jump — 로봇찌빠 무한점프
**경로**: `theme/robot_jump/index.html`  
**대상**: 4세+  
**설명**: 위로 무한 올라가는 차징 점프 플랫폼 게임. 폰 세로 비율 (360×640 기준)

**핵심 메커니즘 — 차징 점프**:
- 화면 꾹 누르고 있는 동안 → 로봇 몸이 쪼그라들며 파워 차징 (게이지 바 표시)
- 손 떼는 순간 → 차징량에 비례해 점프 높이 결정 (짧게 탭=낮게, 길게 홀드=높게)
- 최대 차징 시 2배 높이, 오버차징 없음 (상한선 고정)

**화면 스크롤 방식**:
- 로봇이 화면 상단 1/3 지점을 넘으면 카메라가 위로 따라옴
- 아래로 내려가면 카메라는 그 자리 유지 → 로봇이 화면 아래로 사라지면 게임오버
- 멈춤 없음 — 판 위에 서있어도 카메라가 아주 천천히 계속 아래로 밀어냄 (압박감)

**판 종류 (5가지)**:

| 종류 | 색상 | 효과 |
|------|------|------|
| 고정 판 | 회색/흰색 | 일반 착지 |
| 이동 판 | 파란색 | 좌우로 왕복 이동 |
| 사라지는 판 | 주황색 | 밟으면 2초 후 사라짐 (깜빡임 경고) |
| 컨베이어 벨트 판 | 노란색+화살표 애니메이션 | 착지 시 로봇을 왼쪽 또는 오른쪽으로 밀어냄 (방향은 판 생성 시 랜덤 결정, 화살표로 방향 표시) |
| 스프링 판 | 초록색+스프링 SVG | 밟으면 일반 점프의 2.5배 높이로 튀어오름, 스프링 압축→해제 애니메이션 |

**컨베이어 벨트 상세**:
- 로봇이 판 위에 서있는 동안 매 프레임 x += ±2px 강제 이동
- 판 위 화살표 패턴이 스크롤 애니메이션 (CSS `@keyframes` 로 좌/우 흐름)
- 좌우 방향은 판 생성 시 50% 확률로 결정, `dir: 'left'|'right'` 속성 저장
- 화면 밖으로 밀려나지 않도록 x 범위 클램프

**스프링 판 상세**:
- 착지 감지 시 `velocityY = -jumpPower * 2.5` 즉시 적용
- 스프링 SVG: 판 중앙에 코일 모양 (압축 시 scaleY 0.3 → 해제 시 1.2 bounce)
- 효과음 트리거 포인트: 스프링 해제 순간

**판 구성 (구간별 비율)**:

| 구간 | 고정 | 이동 | 사라짐 | 컨베이어 | 스프링 |
|------|------|------|--------|---------|--------|
| 0~500 | 70% | 20% | — | 5% | 5% |
| 500~1500 | 40% | 30% | 10% | 10% | 10% |
| 1500~3000 | 20% | 30% | 20% | 20% | 10% |
| 3000+ | 10% | 25% | 25% | 25% | 15% |

**판 너비**: 초반 120px → 후반 60px으로 점점 좁아짐  
**판 이동 속도**: 초반 1px/frame → 후반 3px/frame

**로봇 SVG 디자인**:
- 네모난 머리 (안테나 1개, 눈 2개 — 눈이 아래를 봄)
- 몸통 (가슴에 작은 하트 배터리 게이지 표시)
- 짧은 팔 (양옆), 다리 (착지 시 쭈그림 애니메이션)
- 차징 중: 몸이 아래로 scaleY 압축 (0.7)
- 점프 중: 몸이 위로 늘어남 (scaleY 1.2), 팔 위로 올라감
- 착지 시: 발이 쿵 눌리는 bounce

**UI**:
- 좌상단: ← 홈 (`../../index.html`)
- 우상단: 현재 높이(m) — 점수
- 우측 중간: 세로 차징 게이지 바 (꾹 누를 때 채워짐, 파랑→노랑→빨강)
- 최고 기록 (베스트 스코어) localStorage 저장

**게임오버**:
- 로봇이 화면 하단 밖으로 사라지면 즉시 게임오버
- 회전하며 떨어지는 애니메이션
- 결과 화면: 도달 높이 + 최고 기록 + 다시하기 + 홈

**기술 구현**:
- requestAnimationFrame 게임루프
- 판은 배열로 관리, 화면 위로 사라진 판 제거 + 위에서 새 판 생성
- 로봇 물리: velocityY (중력 0.4px/frame 가속), 판 충돌은 위에서 내려올 때만 감지
